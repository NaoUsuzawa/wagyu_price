# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
- リーモートリポジトリ： ネット上に配置して複数人で共有するためのリポジトリ
- ローカルリポジトリ： 開発者一人がひとりが使用するために自分のPC上に配置するリポジトリ


## プッシュとマージの違いは何でしょうか？
- プッシュ： ローカルリポジトリの変更内容をリモートリポジトリに送る操作
- マージ： 現在いるブランチに指定したブランチの変更流歴を取り組む操作


## コミットとプッシュの違い
- コミット：ローカルリポジトリに変更履歴を保存する操作
- プッシュ：ローカルで作成したコミットをリモートリポジトリに送信する操作のため、チームメンバーがその変更を見られるようになる


## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
- 何を変更したのかが一目で分かる内容にする


## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
- ローカルでマージ： 自分のパソコン上（ローカルリポジトリ）でブランチ同士をマージし、その結果をリモートリポジトリにPushする方法。他の人のレビューを挟まずにマージできるため、主に個人開発や簡単な修正で使われる。
- プルリクエストでのマージ： 作業ブランチの変更内容をGitHub上でプルリクエストとして提出し、変更内容を確認・レビューしてもらった後にmasterブランチへマージする方法。チーム開発で多く使われ、ミスやバグを事前に防ぐことができる。



## コンフリクトを起こしてしまった場合、どう対処すべきですか？
- 不要なコードを削除し、必要なコードだけを残す。分からない場合は、チームメンバーと相談しながら作業を進める。修正後に再度コミットする。
